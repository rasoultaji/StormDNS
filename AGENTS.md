# StormDNS — Agent Guide

## Target Network

This is a censorship-circumvention tool for countries (notably Iran) where the national firewall
applies deep packet inspection, active probing, and protocol whitelisting. DNS is often the only
unblocked channel. The network is **slow, lossy, and unstable** — loading a single page can take
minutes. **Every design decision assumes this environment:**

- **Bandwidth is the bottleneck.** Compression (ZSTD/LZ4/ZLIB), minimal packet overhead (~5–7 B),
  and DNS-safe encoding exist to squeeze every usable byte through the DNS query/response limit.
- **Packet loss is the norm, not the exception.** ARQ (send/recv windows, RTO, NACK) is mandatory
  — there is no TCP fallback; everything tunnels over UDP/53.
- **MTU is critical.** DNS resolver limits (~250 B payload) require active MTU discovery
  (`internal/client/mtu.go`). Never assume you can send large frames.
- **Latency is high and unpredictable.** Multi-path balancing, resolver health checks,
  auto-failover, and local DNS caching exist because resolvers die or slow down frequently.
- **Every byte of protocol overhead matters.** Avoid adding fields, handshakes, or round-trips
  that inflate DNS query sizes.

## Build & Test

```bash
go build ./cmd/client && go build ./cmd/server
go vet ./...
go test ./...                          # all packages
go test -v -run TestName ./internal/pkg  # single test
go test -race ./internal/pkg/...       # with race detector
```

No linter/formatter config beyond `go vet`. No task runner, Makefile, or pre-commit hooks.

## Architecture

| Path | Purpose |
|------|---------|
| `cmd/client/main.go` | Client entrypoint (SOCKS5/4 proxy, DNS tunnel) |
| `cmd/server/main.go` | Server entrypoint (UDP DNS listener, SOCKS5 upstream) |
| `internal/client/` | Client logic (balancer, dispatcher, resolver health, session, MTU, SOCKS, TCP stream, DNS listener) |
| `internal/udpserver/` | Server runtime (session mgmt, DNS tunnel, deferred processing) |
| `internal/vpnproto/` | Custom VPN protocol (packet build/parse, payload, control blocks) |
| `internal/security/` | Encryption: AES-GCM, ChaCha20, XOR — key file generated by server on first run |
| `internal/arq/` | Automatic Repeat-reQuest (send/recv windows, RTO, NACK) |
| `internal/basecodec/` | DNS-safe encoding (lowerbase32, lowerbase36, rawbase64) |
| `internal/config/` | TOML config loading (`BurntSushi/toml`) — no env vars |
| `internal/dnsparser/` | DNS packet parsing and response handling |
| `internal/compression/` | ZSTD / LZ4 / ZLIB (via `klauspost/compress`, `pierrec/lz4`) |
| `internal/logger/` | Colored console logging (Win/Unix) |
| `internal/version/` | Build version injected via `-ldflags` |
| `scripts/bench/bench.go` | Integration benchmark (spawns local server+client, measures throughput) |

## Config

- TOML only. No env vars. Samples: `client_config.toml.simple`, `server_config.toml.simple`
- Config paths resolved relative to executable via `internal/runtimepath.Resolve()`
- Encryption key: run server once to generate `encrypt_key.txt`

## Testing Notes

- 48 test files across 16 packages. Standard `testing` package (no external test framework).
- `udpserver` tests cover DNS tunnel capacity, session cleanup, stream SYN, SOCKS5 watchdog.
- `client` tests cover balancer, dispatcher, session init, resolver health, MTU, SOCKS manager, TCP stream, tunnel runtime, async runtime, ping manager.
- No integration/end-to-end tests in CI (bench tool is standalone).
- Some tests may need `-timeout` flag if run under race detector.
